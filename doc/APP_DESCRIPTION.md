## 1. Целевая аудитория и UX-принципы

### Аудитория

Приложение рассчитано на **широкую возрастную категорию: 14–80 лет**.

| Группа | Особенности | Решение |
|--------|-------------|---------|
| Школьники/студенты | Привыкли к мобильным приложениям, хотят скорость | Горячие клавиши, быстрые переходы |
| Взрослые 30–50 | Основные пользователи транспорта, ценят информативность | Крупные блоки, чёткая иерархия |
| Пожилые 60+ | Слабое зрение, не уверены в технологиях | Шрифт 14px+ минимум, крупные касания 44px |

### Ключевые UX-принципы

**Принцип трёх касаний** — любая информация доступна за 1–3 касания от главного экрана.

**Информация без поиска** — нижняя панель показывает ближайшие остановки и пробки без каких-либо действий со стороны пользователя.

**Контекстные предупреждения** — если маршрут задержан, предупреждение появляется автоматически на экранах маршрута и остановки — пользователю не нужно искать эту информацию.

**Деградация без сети** — при отсутствии Интернета приложение работает полностью на статических данных GTFS без потери базовой функциональности.

## 3. Загрузочный экран

### Что отображается (React)

Тёмный экран (градиент) с:
- Название приложения на выбранном языке (Камчатка.Транспорт / Kamchatka Transit / …)
- Три фичи: реалтайм, расписание, офлайн
- Четыре кнопки языка: **SVG-флаг слева + подпись справа** (ru, en, zh, ja)
- Кнопка «Продолжить»

После выбора языка — загрузка соответствующего GTFS ZIP (прогресс в консоли `[GTFS]`).

### Последовательность загрузки (React)

```
1. React mount → App.jsx
2. splash? → Splash (выбор языка) или сразу карта
3. loadGtfsFeed(lang) → fetch ZIP → gtfs.worker.js
4. setGtfsData() → Zustand store
5. refreshFeedsInBackground() (через 3 сек)
6. startVehicleTracker(wsUrl) (через 1 сек) → WS или симулятор
7. precacheRegionTiles() — предзагрузка тайлов (фон)
```

Legacy (`app.html`): `loadGTFS()` → `hideLd()` → `rtStartSim()`.

### Обработка ошибок

При ошибке загрузки данных появляется встроенная заглушка с понятным текстом на русском, без технических деталей для пользователя.

## 4. Экран: Карта

### Верхняя панель (React, TopBar)

```
┌────────────────────────────────────────────┐
│ [🌤 −2°]  [🚌 12 ✕]  [Ост. «Центр» ✕]     │  ← glass-плашки
└────────────────────────────────────────────┘
```

- **Погода** — всегда (если API настроен); тап скрывает/показывает
- **RouteChip** — при фокусе маршрута; ✕ сбрасывает
- **StopChip** — при фокусе остановки; ✕ сбрасывает

### Визуальные компоненты (legacy-референс)

```
┌────────────────────────────────┐
│ [🛰 Симулятор]    [≡ Слои]    │  ← legacy: верхняя панель
│                                │
│                                │
│         КАРТА (deck.gl)        │  ← WebGL полноэкранно
│                                │
│     ·  ·  ·   ·  ·            │  ← Точки остановок
│        /~~~~~\                 │
│       / Марш.  \               │  ← Линия маршрута + анимация
│      ·           ·             │
│           ⊙  ⊙ ⊙              │  ← Маркеры ТС (движутся)
│     🚗 ⚠               ┌──┐   │
│             🚧          │＋│   │  ← FAB зум
│                         │－│   │
│                         │◎│   │  ← FAB геолокация
│                                │
│ ┌────────────────────────────┐ │
│ │ ⚠ 2 ДТП       [▲ тянуть] │ │  ← Умная панель
│ └────────────────────────────┘ │
│ [🗺 Карта] [🚌 Маршруты] [🔀] │  ← Навбар
└────────────────────────────────┘
```

### Слои карты (включаются/выключаются)

| Слой | По умолчанию | Описание |
|------|-------------|----------|
| 🛑 Остановки | Вкл | Серые точки, цветные при активном маршруте |
| 🚌 Маршруты | Вкл | Линия shape + анимированный след TripsLayer |
| ⚠️ Дорожные события | Вкл | Пульсирующие маркеры по координатам событий |

### Поведение карты

**При выборе маршрута (React)**: одна линия shape, остановки и ТС только этого маршрута; RouteChip вверху; камера к трассе.

**При фокусе остановки (React)**: все линии через остановку, связанные остановки и ТС; StopChip вверху; камера центрируется на остановке.

**Legacy**: при открытии остановки камера может не перемещаться.

**Двойной тап**: приближение на +1 зум уровень.

**Pinch**: нативный зум deck.gl.

**Геолокация**: при первом запуске запрашивает разрешение. Синяя точка с пульсирующим кольцом. При движении обновляется плавно.

## 11. Система реалтайм-индикаторов

### RT-бейдж (верхний левый угол)

| Состояние | Визуал | Текст |
|-----------|--------|-------|
| Офлайн | ⚫ серая точка | Офлайн |
| Подключение | 🟡 мигающая точка | Подключение… |
| Симулятор | 🟢 пульсирующая точка | Симулятор |
| Live WS | 🟢 пульсирующая точка | Live |
| Ошибка | 🔴 статичная точка | Ошибка |

### RT-панель (при тапе на бейдж)

```
🛰 Реалтайм

Соединение          Симулятор
Последнее событие   09:14:32
ETA-обновлений      47
Позиций ТС          5
Режим               Симулятор

[ ws://localhost:3000 ] [Подключить]
[⏸ Стоп] [🗑 Сброс ETA]
```

### ETA в экране остановки

Зелёная строка над временами маршрута:
```
🟢 · Реалтайм ETA        3   мин ● LIVE
```

Первое время в строке подсвечивается зелёным с точкой.

## 12. Дизайн-система

### Цветовая палитра

```

Нефритовый          #059669  → свободно, онлайн, прибытие, доступность МГН
Рубиновый           #DC2626  → ДТП, переполнен, опоздание, ошибки
```

## 13. Все варианты использования (Use Cases)

### UC-01: Узнать когда придёт автобус

```
Актор: Пассажир на остановке
Предусловие: Приложение открыто

Основной сценарий:
1. Пользователь видит карту
2. Тапает на точку ближайшей остановки
3. Открывается экран остановки
4. Видит все маршруты с ближайшими временами
5. Первое будущее время выделено крупно и янтарно

Альтернатива A (WebSocket активен):
3a. Над временами появляется зелёная строка «Реалтайм ETA · X мин»

Альтернатива B (нет геолокации):
2b. Тянет нижнюю панель → находит остановку в списке → тапает
```

### UC-02: Проверить задержан ли маршрут

```
Актор: Пассажир, ожидающий автобус
Предусловие: Приложение открыто

Основной сценарий:
1. Открывает экран конкретного маршрута
2. Видит красный/янтарный баннер с описанием события
3. Тапает на баннер → открывается модалка события
4. Читает детали и затронутые маршруты

Альтернатива (через остановку):
1а. Открывает экран остановки
2а. Видит янтарное предупреждение в верхней части
```

### UC-03: Узнать как добраться из А в Б

```
Актор: Новый пассажир
Предусловие: GTFS данные загружены

Основной сценарий:
1. Тапает «🔀» в нижнем навбаре
2. Выбирает точку «Откуда» на карте или старт от текущей геопозиции
3. Выбирает точку «Куда» на карте
4. Нажимает «🔍 Найти маршрут»
5. Видит лучший маршрут по сети: подход к остановке, поездка, пересадки и финальный проход
6. Видит суммарное время, число пересадок и сегменты маршрута на карте

Маршрут не найден:
6a. Показывается сообщение, что подходящих связей в пределах ближайших остановок не найдено
```

### UC-04: Узнать удобства в автобусе

```
Актор: Пассажир с потребностью (кресло-коляска, разряженный телефон)
Предусловие: На карте виден автобус

Основной сценарий:
1. Тапает на иконку движущегося автобуса на карте
2. Открывается карточка ТС
3. Видит блок «Удобства на борту» — все 8 опций со статусом ✓/✗
4. Видит блок «Доступность для МГН» с явным статусом
5. Видит % заполненности и количество свободных мест

Интерпретация:
- Зелёный «Свободно» → можно садиться с коляской
- «USB ✗» → лучше дождаться следующего
```

### UC-05: Мониторинг дорожной обстановки

```
Актор: Пассажир, планирующий поездку
Предусловие: Приложение открыто

Основной сценарий:
1. На карте видны красные/янтарные маркеры событий
2. Тапает на маркер → карточка события
3. ИЛИ тянет нижнюю панель → горизонтальный скролл событий
4. Читает: «ДТП на ул. Ленинской, задержки ~8 мин, маршруты 3, 11, 14»
5. Принимает решение: ждать или воспользоваться другим маршрутом
```

### UC-06: Сохранить любимый маршрут

```
Актор: Постоянный пассажир
Предусловие: Открыт экран маршрута или список

Основной сценарий:
1. Открывает экран маршрута
2. Тапает «☆ В избранное»
3. Кнопка становится «★ В избранном»
4. При следующем открытии «Маршруты» → маршрут вверху списка
```

### UC-07: Работа без интернета

```
Актор: Пассажир в зоне без сети
Предусловие: Приложение ранее открывалось

Основной сценарий:
1. Открывает приложение (нет сети)
2. Service Worker отдаёт кэшированный HTML
3. GTFS данные встроены в HTML → загружаются из base64
4. Карта работает (тайлы кэшированы SW)
5. Расписание показывает статические данные
6. RT-бейдж показывает «Офлайн»
7. Симулятор отключён (нет смысла без реальных данных)
```

### UC-08: Установить приложение на телефон

```
Актор: Регулярный пользователь
Предусловие: Chrome на Android/iOS, открыт сайт

Основной сценарий (Android):
1. Открывает сайт в Chrome

3. Тапает «Установить»
4. Приложение появляется на рабочем столе

iOS Safari:
2b. Тапает «Поделиться» → «На экран «Домой»»
```

### UC-09: Подключить реальный WebSocket-сервер

```
Актор: Администратор/оператор
Предусловие: Есть работающий WS-сервер

Основной сценарий:
1. Тапает на RT-бейдж (верхний левый)
2. Открывается RT-панель
3. Вводит адрес: ws://server:3000
4. Нажимает «Подключить»
5. Симулятор останавливается
6. Реальные ETA начинают поступать
7. Бейдж становится «🟢 Live»
```
