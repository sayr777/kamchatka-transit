# Реализация мультиязычности в Transit PWA

## Цели
- Базовые языки интерфейса: русский и английский
- Расширенные языки: китайский и японский
- Отдельный культурный уровень: корякский и ительменский для локальных инициатив

## Приоритет внедрения
1. **P0**: RU + EN для всего UI shell
2. **P1**: ZH + JA для основных экранов, ошибок и модальных окон
3. **P2**: поддержка локальных языков для приветственных экранов и культурных разделов

## Что переводить
- названия кнопок
- системные сообщения
- пустые состояния
- карточки автобусов и остановок
- настройки и экран выбора фида

## Что не переводить автоматически
- названия остановок из GTFS
- route_id, route_short_name, route_long_name
- provider / agency names

## Технический план
- выделить `public/i18n/ru.json`, `en.json`, `zh.json`, `ja.json`
- создать модуль `public/js/i18n.js`
- использовать функцию `t(key)` во всех слоях UI
- язык хранить в `localStorage`
- fallback цепочка: selected → en → key

## Пример ключей
```json
{
  "app.title": "Транспортный портал",
  "map.where_am_i": "Где я?",
  "feed.load_zip": "Загрузить ZIP",
  "stop_board.no_service": "На выбранную дату рейсов нет"
}
```

## UX рекомендации
- показывать переключатель языка в шапке
- не менять язык GTFS-данных насильно
- добавлять мультиязычные подсказки для туристов на первом экране
