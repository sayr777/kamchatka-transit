## 3. Типографика

### Шрифтовые пары

| Роль | Шрифт | Файл |
|------|-------|------|
| **Заголовки, кнопки, номера маршрутов** | `Unbounded` (700 / 600) | `vendor/fonts/f36–f40.woff2` |
| **Основной текст, UI-элементы** | `Golos Text` (400 / 500 / 600) | `vendor/fonts/f1.woff2` |
| **Запасной** | `system-ui, sans-serif` | — |

### Переменные

```css
--head: 'Unbounded', system-ui, sans-serif;
--body: 'Golos Text', system-ui, sans-serif;
```

### Шкала размеров

| Токен | Размер | Применение |
|-------|--------|------------|
| `--fs-xl` | `22px` | Заголовки панелей, названия маршрутов |
| `--fs-lg` | `17px` | Подзаголовки, названия остановок |
| `--fs-md` | `15px` | Основной текст, кнопки |
| `--fs-sm` | `13px` | Вторичный текст, метки |
| `--fs-xs` | `11px` | Капшны, бейджи, временны́е метки |
| `--lh` | `1.5` | Line-height для основного текста |

### Правила применения
- `--head` (`Unbounded`) — только для коротких строк: заголовки, кнопки, номера «№2», «№23»
- `--body` (`Golos Text`) — весь длинный текст: описания, расписание, поиск
- Не использовать `Unbounded` для текста длиннее 4–5 слов подряд
- Минимальный контраст текста — **4.5 : 1** (WCAG AA)

## 6. Анимации и переходы

```css
--ease: .18s cubic-bezier(.4, 0, .2, 1);   /* Material Design standard easing */
```

| Состояние | Поведение |
|-----------|-----------|
| **hover** | `translateY(-1px)` + тень усиливается |
| **active / tap** | `scale(.94)` + тень возвращается к базовой |
| **disabled** | `opacity: .4`, `cursor: default`, тень убирается |
| **focus-visible** | `outline: 2px solid var(--acc); outline-offset: 2px` |

Все интерактивные элементы используют единый `transition: … var(--ease)`.

## 7. Гармонизация пользовательского интерфейса (UI)

Гармонизация UI — это процесс создания удобного, эстетичного и интуитивно понятного интерфейса, который помогает пользователям (от 14 до 80 лет) быстро и без лишних усилий достигать своих целей.

### Основные принципы гармонизации UI в «Камчатка.Транспорт»

**1. Простота и минимализм**  
Чем меньше элементов на экране — тем лучше.  
→ Используем прогрессивное раскрытие: дополнительные опции (полное расписание, фильтры) появляются только по запросу пользователя.

**2. Последовательность и консистентность**  
Пользователь не должен переучиваться.  
→ Кнопка «Назад» всегда в левом верхнем углу.  
→ Все маршруты отображаются в едином стиле бейджей.  
→ RT-индикатор всегда в одном месте (верхний левый угол карты).

**3. Визуальная иерархия**  
Важное видно сразу.  
→ F-паттерн чтения: крупный номер маршрута → направление → ETA (самое важное).  
→ Главные действия (Найти маршрут, На карту) выделяются размером и акцентным цветом.

**4. Баланс текста и визуальных элементов**  
Текст лаконичен, иконки и иллюстрации дополняют его.  
→ Названия остановок — крупно и жирно.  
→ Времена прибытия — максимально заметны.  
→ Минималистичный вулкан используется только как бренд-символ.

**5. Обратная связь**  
Мгновенная реакция на каждое действие.  
→ Нажатие кнопки: изменение цвета + scale-эффект (250ms).  
→ Появление реалтайм ETA: зелёная подсветка + лёгкая анимация.  
→ Тактильная обратная связь (вибрация) на мобильных устройствах.

**6. Адаптивность и отзывчивость**  
Приложение отлично работает на всех устройствах — от бюджетных смартфонов до больших экранов.  
→ Используем CSS Custom Properties + `viewport-fit=cover`.  
→ Touch-target минимум 44px.

**7. Доступность (Accessibility)**
- Контраст текста не ниже 4.5 : 1 (WCAG AA)
- Крупные целевые области (touch-target 44 × 44 px)
- Поддержка screen readers (ARIA-атрибуты)
- Увеличение шрифта до 200 % без потери функциональности

### Инструменты

| Инструмент | Назначение |
|-----------|------------|
| **Figma** | Дизайн-система и прототипы |
| **Chrome DevTools** | Тестирование адаптивности |
| **Lighthouse** | Аудит доступности и производительности |

### Типичные ошибки, которых мы избегаем

| Ошибка | Как мы решаем |
|--------|--------------|
| Сложная навигация | Максимум 3 уровня вложенности |
| Перегруженные экраны | Карта остаётся чистой, детали — в панелях |
| Нарушение платформенных гайдлайнов | PWA-поведение, safe-area, viewport-fit |
| Отсутствие визуальной иерархии | F-паттерн, размер и цвет выражают важность |
| Игнорирование пожилой аудитории | Шрифт от 13px, крупные кнопки, простые жесты |

## 9. Компоненты

### Плавающие кнопки (FAB)

```
Размер: 44 × 44 px
Скругление: --r (12px)
Фон: var(--panel)
Тень: var(--sh)
Иконка: 17–18px, цвет var(--t2)
Активная: фон var(--acc), цвет #fff
```

**Стек справа (сверху вниз):**
1. Компас `N` — `top: calc(12px + env(safe-area-inset-top))`
2. Поиск `🔍` — `top: calc(60px + env(safe-area-inset-top))`
3. Планировщик `⇄` — `top: calc(108px + env(safe-area-inset-top))`
4. Тема `☽` — `bottom: calc(80px + env(safe-area-inset-bottom))`
5. Обновление `⇄` — `bottom: calc(128px + env(safe-area-inset-bottom))`
6. Геолокация `⊕` — `bottom: calc(32px + env(safe-area-inset-bottom))`

### Кнопка CTA (основное действие)

```
Высота: 52px (padding 17px)
Ширина: 100% / max-width
Шрифт: --head, 15–16px, 700
Фон: gradient #FC3F1D → #FF6B4A
Скругление: 14px
Тень: 0 6px 24px rgba(252,63,29,.38)
```

### Бейдж маршрута

```
Фон: цвет маршрута из GTFS route_color
Текст: route_text_color из GTFS
Шрифт: --head, 11px, 700
Padding: 4px 12px 4px 9px
Скругление: --r-sm (8px)
```

### Поиск (search bar)

```
Свёрнут: 44 × 44px, иконка лупы
Развёрнут: min(320px, 100vw − 76px)
Анимация ширины: var(--ease)
```

- Небо: градиент `#1a3a6b → #4a8fd4 → #a8d8f0`
- Горизонт/луга: `#c5e8c0 → #4e8a4e → #2d5e2d`

## 11. Карта

| Компонент | Технология |
|-----------|------------|
| Подложка | MapLibre GL + Mapbox raster tiles |
| Слои данных | deck.gl (PathLayer, ScatterplotLayer, TextLayer) |
| Офлайн-тайлы | `src/gtfs/precache.js` → Cache API |

**Остановки:** точки ScatterplotLayer (zoom ≥ 13)  
**ТС:** цветная точка + halo + TextLayer с номером маршрута  
**Маршрут (выбранный):** PathLayer, цвет из `route_color` GTFS

> Legacy `app.html` использовал CartoDB/OSM tiles. React-версия — кастомный стиль Mapbox.

## 12. 10 лайфхаков мобильного UX/UI

Практические принципы, применённые в «Камчатка.Транспорт» на основе лучших практик мобильного дизайна.

---

**1. Упрощение интерфейса**  
Минимализм — основа. Карта остаётся чистой, панели с деталями появляются только по действию пользователя.  
→ На главном экране только карта + 6 FAB-кнопок. Никаких лишних элементов.

---

**2. Консистентность навигации**  
Пользователь не должен думать, где что находится.  
→ Компас, поиск, планировщик — всегда справа сверху. Геолокация, тема, обновление — справа снизу. Никогда не меняются местами.

---

**3. Пальцеориентированный дизайн**  
Все цели — крупные и доступные большому пальцу.  
→ Минимальный touch-target **44 × 44 px** (Apple HIG). FAB-кнопки на правой стороне — в зоне большого пальца правой руки.

---

**4. Оптимизация форм**  
Поля ввода сведены к минимуму.  
→ Поиск остановки — одно поле, автосаджест по мере ввода. Никаких лишних шагов. Дата/время в попапе остановки — редактируемые, но заполнены текущим значением по умолчанию.

---

**5. Адаптивный и реактивный дизайн**  
Работает на любом телефоне — от бюджетного Android до iPhone с Dynamic Island.  
→ `viewport-fit=cover`, `env(safe-area-inset-*)` для всех отступов. CSS Custom Properties адаптируют тему без перезагрузки.

---

**6. Скорость загрузки**  
Приложение работает офлайн после первого запуска.  
→ GTFS парсится в Web Worker. Карта — WebGL (deck.gl). Code splitting: отдельные chunks для maplibre, deckgl, react. Шрифты: Google Fonts (Golos Text, Unbounded) в React; локальные woff2 в legacy `vendor/fonts/`.

---

**7. Интуитивные жесты**  
Жесты соответствуют платформенным стандартам.  
→ Свайп вверх — раскрыть нижний лист с остановками. Щипок — зум карты (нативный deck.gl). Тап на маркер — попап с расписанием.

---

**8. Тёмный режим / режим чтения**  
React: автоматически по `prefers-color-scheme` (`src/index.css`).  
Legacy: ручной переключатель `☽/☀` + CartoDB Dark Matter tiles.  
Панели: `#FFFFFF` / `#1C1C1E` (свет) · `#2C2C2E` (тём).

---

**9. Обратная связь на действия**  
Каждое нажатие даёт немедленный отклик.  
→ Кнопки: `scale(.94)` + изменение тени за `0.18s`. Загрузка GTFS: прогресс-бар с текстовым статусом. Появление ETA: зелёная подсветка `.ok` цветом `#34C759`.  
→ На поддерживающих устройствах: `navigator.vibrate(10)` при тапе на маркер.

---

**10. Прогрессивный онбординг**  
Пользователь не перегружается информацией сразу.  
→ Первый запуск: только сплэш с выбором языка → загрузка → карта. Никаких туториалов, оверлеев, обязательной регистрации. Функции планировщика и фильтров открываются по мере необходимости.

---

## 13. Антипаттерны — чего не делать

- ❌ Белый фон заставки — только пейзаж
- ❌ Один шрифт для всего — заголовки и текст разные
- ❌ Тени без токена — только через `--sh` / `--sh-up`
- ❌ touch-target меньше 44px
- ❌ Более трёх основных цветов одновременно на экране
- ❌ Анимации дольше 300ms для UI-переходов
